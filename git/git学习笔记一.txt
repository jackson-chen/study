本文件主要是记录学习laravel框架中用到的git知识

【git是什么呢？】
git是一个分布式代码控制系统，可以对项目进行版本控制，比如追踪代码修改，进行版本回退等等操作


【配置用户名和邮箱】
使用如下两个命令设置用户名和邮箱：
git  config --global  user.name xiao-xin
git  cofnig --global  user.email mysoftwareer@gmail.com

使用如下命令查看我们配置的全局设置:
git config --global --list
如果不对git做全局设置，使用上面这个命令会提示/home/vagrant/.gitconfig文件不存在。只有在配置了全局设置才可以使用这个命令，当然你也可以到这个文件去查看全局配置信息，这和使用命令查看是一样的效果


【设置git推送分支】
git config --global push.default simple

设置git命令push的默认模式为简单模式，当我们使用git push时没有指定分支，会把当前分支推送出去，而不是报错

【git初始化】
git  init 
在项目根目录使用这个命令，来完成git的初始化，使git来跟踪管理项目代码

【git add添加修改】
git add -A 
添加所有文件到git暂存区中

对不想用git版本管理的文件，可以配置在.gitignorewen文件中，如何配置我会用单独讲

【git status查看git状态】
git status
使用该命令可以查看哪些文件存放在暂存区中，以未着这些文件还没有提交到git仓库中


【git commit 提交修改】
git commit -m 'initial commit'
将暂存区中的文件提交到git中, -m表示本次提交
的描述


【git log 查看提交记录】
git log
通过该命令可以查看历史提交记录

【git 撤销暂存区中文件修改】
git checkout -f
强制撤销暂存区中更改的文件

【通过ssh访问Github】
1. 需要在Homestead.yaml文件中地方keys配置我们主机中的SSH key私钥到虚拟主机，具体如何配置可以看laravel环境搭建

2.把SSH key添加到ssh-agent

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_rsa

在使用vagrant账户执行 ssh-add ~/.ssh/id_rsa 没有访问文件的权限，切换到root账户下执行ssh-add

3.把本地中的公钥设置到Github令牌管理页面中

4.在Github上创建一个仓库


【本地仓库推送到Github】
添加远程仓库
git remote add origin git@github.com:xiao-xin/weibo.git

把本地的master推送到远程仓库的master,并设置默认的推送仓库
git push -u origin master

【git checkout 切换分支】
git checkout master
将当前分支切换为maseter分支，master是git默认创建的主分支，其他的分支都是从master上衍生出来的

【git checkout -b 创建分支】
git checkout -b static-pages 
创建分支并切换当前分支到新分支

平时的修改操作都是在分支上完成的，修改完后，合并分支会将分支上所有的改动并入到master分支上，如果删除分支，分支上的修改会被销毁，而master分支不会受到影响


【git merge 分支合并】
git merge fake-branch
把当前分支合并到fake-branch之上

【git branch -d 删除分支】
git branch -d fake-branch
可以删除分支，如果分支上有内容没有合并到master分支，会提示是否强制删除
